# AI агрегатор постов в телеграмме
## Описание
Проект представляет собой бота в телеграмме, направленный на фильтрацию постов из разных каналов. Будет реализованы следующие возможности:
- Добавление каналов в подборку
- Фильтрация постов по промту
- Фильтрация дублирующихся постов(например, одинаковая новость из разных каналов)
- Агрегация постов по определённым периодам (например агрегировать все посты в одно сообщение один раз в день)
- Задавать промт для отправки сообщений (например, задать структуру “Новость: …; Из какой страны: …“)
- RAG по отфильтрованным постам, возможность задавать вопросы по тем постам которые прошли начальный фильтр и были сохранены в базу
- Возможность создавать несколько различных подборок

## Реализация

В реализации я стараюсь придерживаться чистой архитектуры. Пара слов о ней в моём проекте.
Я разбиваю на стандартные 4 слоя:
 - Domain: здесь лежат интерфейсы, промпты, схемы структурированных ответов и т.д., которые потом будут использоваться для реализации логики
 - Use Cases: здесь использую всё что объявил в domain, реализую отдельные юз кейсы приложения 
 - Infrastructure: здесь лежит реализация интерфейсов из domain слоя.
 - Presentation: здесь всё что относится к представлению, руты, схемы запросов и ответов и т.д.

В приложении реализовал на текущий момент один эндпоинт который отвечает за классификацию текста. Отправляем текст и промт по которому хотим классифицировать текст. В ответ получаем релевантен ли текст заданному промту. Реализовывал используя бесплатную модель с openrouter.

Пример env файла
```dotenv
OPENROUTER_KEY=xxx-xxx-xxx
OPENROUTER_MODEL=minimax/minimax-m2.5
```
Команда для запуска
```commandline
uvicorn src.telegram_aggregator.main:app --reload
```
Свагер на локалке:
http://127.0.0.1:8000/docs
